3. Sequential HERO-ML

As mentioned, the sequential part of HERO-ML is a minimum, standard WHILE language. It is strongly and explicitly typed, and a HERO-ML program consists of a sequence of type declarations followed by a HERO-ML statement. In more detail, it has the following features:

  • Types: there are simple numerical types int, float, and a boolean type bool. Builtin functions and operators have function types on the form t1 -> t2, where t1 is the argument type and t2 is the result type. If the function takes several arguments then its argument type is a tuple type (t1,...,tn). So, for instance, integer addition has the function type (int,int) -> int. However, HERO-ML is not a general higher order language.
    Type declarations have the format x : t, where x is a program variable and t is a type.
  • Operators and functions: HERO-ML has the usual arithmetic operators (+,-,*,/), relational operators (=,<), boolean operators (&&,||), and boolean negation not. The arithmetic and relational operators are overloaded with respect to the numerical types, so for instance "+" has both the type (int,int) -> int and (float,float) -> float. Implicit type conversion is not permitted: thus, numerical arguments of different type are not allowed for these operators.
  • "Scalar" assignments: these are conventional, sequential assignments. They have the form x = e, where x is a variable of simple type and e is an expression of the same type.
  • Control structures: there is a while loop while b do s, and a conditional statement if b then s else s', where b is a boolean expression and s, s' are statements. (There is also a side-effect free if-function, called as if(b,e,e'), where e, e' are expressions with the same type.)
  • I/O: HERO-ML has a simple model for i/o where the environment provides an input pipe and an output pipe. The statement in t, where t is a type, reads a value of type t from the input pipe and out e, where e is an expression, evaluates e and writes the result to the output pipe.
    There is no way in HERO-ML itself to tie the pipes to more concrete communication channels, such as stdin and stdout in a unix system. Such associations have to be done in the surrounding environment where the HERO-ML program runs.
Excluded constructs: the current version of HERO-ML lacks some features found in full programming languages. This includes user-defined functions and procedures, user-defined types, records, objects, and pointers. It is possible that some of these features will be added in future versions of HERO-ML.

Here is an example of a simple, sequential HERO-ML program:

i : int; x : float:
   i = 1; x = 0.0;
   while i < 10 do
     x = x + in float;
     i = i + 1;
   out x;

This program reads the first ten floating-point numbers from the input pipe, and writes their sum to the output pipe. This is a purely sequential program, with no data parallelism. Note that the syntax is indentation-sensitive, similar to F# and python.
